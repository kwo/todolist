# Task List

Task List is a local-first CLI for managing tasks stored as Markdown files with YAML front matter.

Each task is a plain text file in a directory, making tasks easy to inspect, edit, back up, and version with Git.

## Goals

- store tasks as Markdown files
- use front matter for task metadata
- keep the format human-readable and manually editable
- provide a simple CLI for common task operations
- support machine-readable output with a global `--json` option

## Shared Specifications

### ID Generation

Task IDs should be auto-generated by the CLI in this format:

- `<prefix><hash>`

Where:

- `<prefix>` comes from the tasklist directory config file `.tasks`
- the config file is a flat `key=value` file
- the supported key is `prefix`
- the default prefix is `task-`
- `<hash>` is lowercase Crockford Base32
- `<hash>` is exactly 4 characters
- `<hash>` is derived from a deterministic FNV-1a 64-bit hash

Lowercase rendered alphabet:

- `0123456789abcdefghjkmnpqrstvwxyz`

Excluded characters:

- `i`, `l`, `o`, `u`

The CLI should:

1. build a seed from `title|description|createdAtUnixNano|nonce`
2. hash it with Go `fnv.New64a()`
3. encode the `uint64` hash using Crockford Base32
4. left-pad with `0` if needed
5. keep the last 4 characters
6. render the suffix in lowercase
7. prefix with the configured `prefix` value from `.tasks`
8. if no config is present, use the default prefix `task-`
9. retry with incrementing nonce values on collision

This gives `32^4 = 1,048,576` possible visible IDs.

See `docs/id-generation.md` for the full spec and Go reference implementation.

### Directory Selection

The CLI should read and write task Markdown files from a task directory determined as follows:

1. `-d, --directory` global CLI option
2. `TASKLIST_DIRECTORY` environment variable
3. default `./tasks`

The `-d, --directory` option is global, so it should appear before the subcommand. Example:

```bash
tasklist -d ./work-tasks list
```

The task directory may also contain a `.tasks` configuration file.

The `.tasks` file is a flat `key=value` file and should support:

- `prefix` â€” the prefix used when generating task IDs

If `prefix` is not set, the default prefix is `task-`.
