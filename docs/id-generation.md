# ID Generation

Task IDs should be auto-generated by the CLI.

## Format

Top-level task IDs use:

- `task-<hash>`

Examples:

- `task-7k9m`
- `task-2w8x`
- `task-v3hq`

The hash portion is:

- Crockford Base32
- exactly 4 characters
- rendered in lowercase for filenames and CLI output

Canonical Crockford Base32 alphabet:

- `0123456789ABCDEFGHJKMNPQRSTVWXYZ`

Lowercase rendered alphabet used in task IDs:

- `0123456789abcdefghjkmnpqrstvwxyz`

Excluded characters:

- `i`
- `l`
- `o`
- `u`

This avoids the common human-confusion cases like `1` vs `l` and `0` vs `o`.

This gives `32^4 = 1,048,576` possible visible IDs, which is sufficient for this project.

## Hash algorithm

Use Go's `hash/fnv` package with **FNV-1a 64-bit**:

- constructor: `fnv.New64a()`

This is intentionally not cryptographic. The goal is short, deterministic, human-friendly IDs, not security.

## Seed input

Build a deterministic seed string from task creation inputs:

- `title`
- `description`
- `createdAt` as Unix nanoseconds
- `nonce`

Seed format:

```text
title|description|createdAtUnix|nonce
```

Example:

```text
Buy groceries|Need milk, eggs, and bread.|1773828000000000000|0
```

## Encoding process

1. Build the seed string.
2. Hash it with FNV-1a 64-bit.
3. Encode the resulting `uint64` using the Crockford Base32 alphabet.
4. Left-pad with `0` if needed.
5. Take the last 4 characters.
6. Render the suffix in lowercase.
7. Prefix with `task-`.

This means the visible 4-character suffix is derived from the least significant Crockford Base32 digits of the 64-bit hash value.

## Collision handling

If a generated ID already exists:

1. Retry with incrementing `nonce` values starting at `0`.
2. Regenerate until an unused ID is found.

Recommended behavior:

- try `nonce = 0, 1, 2, ...`
- stop only when a free ID is found

Given the available space and local-first usage, this should be rare.

## Child IDs

If hierarchical child tasks are added later, use:

- `<parentId>.<n>`

Examples:

- `task-7k9m.1`
- `task-7k9m.2`

For now, plain top-level IDs are enough unless parent-child auto-ID generation is explicitly implemented.

## Go reference implementation

```go
package taskid

import (
	"fmt"
	"hash/fnv"
	"strings"
	"time"
)

const alphabet = "0123456789abcdefghjkmnpqrstvwxyz"
const visibleLen = 4
const prefix = "task"

func Generate(title, description string, createdAt time.Time, exists func(string) bool) string {
	for nonce := 0; ; nonce++ {
		seed := buildSeed(title, description, createdAt, nonce)
		hash := fnv64a(seed)
		suffix := crockfordLastN(hash, visibleLen)
		id := prefix + "-" + suffix
		if !exists(id) {
			return id
		}
	}
}

func buildSeed(title, description string, createdAt time.Time, nonce int) string {
	return fmt.Sprintf("%s|%s|%d|%d",
		title,
		description,
		createdAt.UTC().Unix(),
		nonce,
	)
}

func fnv64a(s string) uint64 {
	h := fnv.New64a()
	_, _ = h.Write([]byte(s))
	return h.Sum64()
}

func crockfordLastN(v uint64, n int) string {
	if v == 0 {
		return strings.Repeat("0", n)
	}

	buf := make([]byte, 0, 13) // enough for uint64 in base32
	for v > 0 {
		buf = append(buf, alphabet[v%32])
		v /= 32
	}

	// reverse
	for i, j := 0, len(buf)-1; i < j; i, j = i+1, j-1 {
		buf[i], buf[j] = buf[j], buf[i]
	}

	s := string(buf)
	if len(s) < n {
		s = strings.Repeat("0", n-len(s)) + s
	}
	if len(s) > n {
		s = s[len(s)-n:]
	}
	return s
}
```

## Notes

- IDs should be generated by the CLI by default.
- Manual overrides may be supported later, but generated IDs should remain the default path.
- Because filenames are `<id>.md`, the generated ID must remain filesystem-safe. Lowercase Crockford Base32 satisfies that requirement.
- The reference implementation hardcodes the prefix as `task-`. Configurable prefixes via `.tasks` are introduced in `user-story-directory-config.md`.
